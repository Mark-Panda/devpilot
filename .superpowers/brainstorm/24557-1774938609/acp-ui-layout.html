ACP 执行页 / 日志页：布局方向

选择你更想要的“信息组织方式”。后续我会按选项继续细化交互与样式。

A

紧凑型（单列）

信息密度高，适合熟练用户；在同一页快速扫到输入、进度、结果与错误。

预览：单列紧凑
任务
规则链 / 节点 / 会话标识（可复制）
输入
Prompt（可折叠/展开）
运行状态
round 3/20 · stopReason=…
流式输出
实时滚动 · 支持“只看尾部/暂停滚动/复制”
stderr / 诊断
stderr tail · 解析丢弃原因（verbose）

优点

  • 一屏内闭环
  • 切换成本低
  • 适合“只盯输出”场景

缺点

  • 新手信息噪声更大
  • 诊断与控制容易被淹没
B

信息分层型（主区 + 侧栏）

把“流式输出”作为主角，控制/诊断/元信息放侧栏；更清晰、更适合长会话。

预览：主区 + 侧栏
流式输出（主区）
实时滚动 · 分段 · 搜索 · 复制
状态
round 3/20 · running
任务信息
规则 / 节点 / cwd（可复制）
诊断
stderr tail（折叠）
交互
ask_question / after_round 弹窗入口与历史

优点

  • 长输出可读性更好
  • 控制与诊断更聚焦
  • 更适合“会话式”交互

缺点

  • 占用更多横向空间
  • 需要更明确的信息架构
