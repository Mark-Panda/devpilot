# ACP UI（方案 B2 + C2）Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在 RuleGo 执行日志/执行页中，把 Cursor ACP 输出区升级为「主区 + 侧栏」布局，并加入分组工具条（搜索/复制/滚动跟随），同时保持全局待处理弹窗（续聊/提问）体验不倒退。

**Architecture:** 以 `CursorACPExecutionDetailSection` 为核心改造点：主区展示流式/最终文本，侧栏聚合交互入口、状态与诊断；工具条以纯前端状态实现，不引入新依赖。全局弹窗 `CursorACPAfterRoundHost` 保持现有堆叠卡片方式，仅做必要的小增强（若需要）。

**Tech Stack:** React + TypeScript（Vite），现有 CSS（`frontend/src/styles/globals.css`），现有渲染组件 `LogTextPreview`。

### Task 2：实现 C2 分组工具条（阅读 / 查找 / 导出）

**Files:**
- Modify: `frontend/src/modules/rulego/CursorACPExecutionDetailSection.tsx`
- Modify: `frontend/src/styles/globals.css`

- [ ] **Step 1: 阅读组**
  - **暂停滚动**：冻结“自动跟随到底部”的行为（不影响用户手动滚动）
  - **只看尾部**：一键滚动到底部，并开启跟随（直到用户暂停）

- [ ] **Step 2: 查找组（搜索输出）**
  - 本地搜索 `bodyText`：
    - 输入框（支持 `/` 快捷键聚焦，Esc 清空/失焦）
    - 上一个/下一个匹配
    - 只做轻量高亮：不改 Markdown 渲染树，采用“在纯文本模式下高亮”或“显示匹配计数 + 跳转”二选一
  - 验收：长文本下不卡顿（必要时对超长文本降级为仅计数/仅跳转）

- [ ] **Step 3: 导出组（复制）**
  - **复制：本轮**（优先复制 `cursor_acp_last_stream_text`，否则复制当前 `bodyText`）
  - **复制：全部**（复制 `bodyText`）
  - 使用 `navigator.clipboard.writeText`，失败时降级为 `textarea` 选中复制

- [ ] **Step 4: 手动验证**
  - 在流式预览不断更新时，开启/关闭跟随不抖动
  - 搜索跳转正常、复制正常
